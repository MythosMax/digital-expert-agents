# Harness glossary

| Term | Meaning |
|---|---|
| Conductor | Coding-agent lead that plans and integrates; owns durable status |
| Part harness | Per-surface control doc under `docs/harness/parts/` |
| Specialist | Domain digital expert (Credit / Legal / Ops) |
| Planner / Supervisor | Orchestrator that decomposes and assigns tasks |
| Canonical event | Runtime-neutral stream event (not vendor-specific) |
| Agent bridge | NestJS `/v1/agent/*` surface for tool backends |
| HITL | Human-in-the-loop durable approval checkpoint |
| Lane | tiny / normal / high-risk intake classification |
| Proof | Evidence that acceptance criteria passed |
| Trace | Durable record of what an agent did on a task |
| Corpus | Domain knowledge collection for RAG |
| Allowlist | Explicit set of tools an agent may call |
| Draft | Reversible proposed action; not executed |
| Eval case | Benchmark or regression scenario with expected checks |
